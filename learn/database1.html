Chapter 5

  • Explain constraints on relational database
  • Superkey
    • Attribute set which is able to uniquely identify a tuple
    • May contain redundant database
  • Key
    • Attribute set which is able to uniquely identify a tuple
    • if we remove any attribute, it is no longer a supekey (minimal)
  • Candidate Key
    • Any Key
  • Primary Key
    • selected as the primary among the candidate keys
  • Unique Key
    • Any other candidate key not selected as Primary
  • Foreign Key
    • Set of attributes which refer to PK
  • Be able to identify PK and FK

BOEINGS ← σ aname = ‘Boeing-777’ (AIRCRAFT)

CERTIFIED_BOEINGS ← CERTIFIED |x| aid = aid BOEINGS

Chapter 9

  • Be able to convert ER diagram to relational schema

  • Step 1: Map regular entity types

    • Create relation for each non-weak entity
    • Only include atomic component attributes
    • Choose one of the keys as primary key
  • Step 2: Map weak entity types
    • Create relation and add all atomic attributes
    • Add PK of owner as FK
    • PK of this table is combo of owner PK and partial key of this table
  • Step 3: Map Binary 1:1 relation types
    • Add PK of one relationship as FK on the other
  • Step 4: Map Binary 1:N relation types
    • Add PK of one-side entity as FK in the other
  • Step 5: Map Binary M:N
    • Create separate relation
    • Add PK of both sides as foreign keys in relation
    • combo of the two PKs will form PK of this table
  • Step 6: Map multivalued attributes (A)
    • create new relation R
    • combination of A and PK of relation forms PK of R
    • If composite, add atomic components as attributes of R
  • Step 7: Map N-ary
    • Create new relation S
    • Add PKs of all participating entity types as FKs
    • the PK of S is combo of those foreign keys (excluding 1-side entity types)

Chapter 14

  • Explain 1nf, 2nf, 3nf, bcnf

    • 1nf: The key (all values must be atomic)

    • 2nf: The whole key (no partial dependencies)

    • 3nf: And Nothing but the key (no transitive dependencies)

    • BCNF: All items with dependencies must be superkey
